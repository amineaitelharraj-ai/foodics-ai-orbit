================================================================================
REPLIT FRONTEND UPDATE: Complex Query Notification
================================================================================

WHAT CHANGED:
The backend now sends a new WebSocket message type when a complex query is
detected (queries that require the supervisor to coordinate multiple agents).

This helps users understand why some queries take longer than others.

================================================================================
NEW WEBSOCKET MESSAGE TYPE
================================================================================

Server -> Client:
{
  "type": "complex_query_notice",
  "message": "This is a complex query that requires analyzing multiple data sources. This may take a bit longer. Feel free to explore our BI tools while you wait!"
}

This message is sent IMMEDIATELY after the router detects a complex query,
BEFORE any "Processing..." or tool events.

================================================================================
IMPLEMENTATION IN src/services/labeeb-api.ts
================================================================================

In the WebSocket onmessage handler, add this case to the switch statement:

```typescript
case 'complex_query_notice':
  console.log('[WS] Complex query detected:', data.message);
  // Option 1: Emit event for component to handle
  if (onComplexQueryNotice) {
    onComplexQueryNotice(data.message);
  }
  // Option 2: Or handle directly here
  break;
```

================================================================================
STATE MANAGEMENT
================================================================================

Add state to track the complex query notice:

```typescript
// In your chat state/context
const [complexQueryNotice, setComplexQueryNotice] = useState<string | null>(null);

// Clear notice when response completes or new message sent
useEffect(() => {
  if (isComplete || isNewMessage) {
    setComplexQueryNotice(null);
  }
}, [isComplete, isNewMessage]);
```

Examples that trigger complex_query_notice:
- "Give me a business overview - how many branches and total sales?"

- "List my categories" (direct to menu agent)
- "Show my orders" (direct to analytics agent)
- "Hello" (direct to general handler)

================================================================================
NEW: LARGE DATA NOTICE
================================================================================

Server -> Client (when list_*/search_* tools are called):
{
  "type": "large_data_notice",
  "message": "Fetching your data... This may take a moment for large datasets. Feel free to explore our BI tools while you wait!"
}

This is sent when tools like list_products_detailed, search_orders_detailed are
called. Only sent ONCE per session (first large data tool).

Add to WebSocket handler:
```typescript
case 'large_data_notice':
  console.log('[WS] Large data fetch:', data.message);
  if (onLargeDataNotice) {
    onLargeDataNotice(data.message);
  }
  break;
```

================================================================================
UPDATED WEBSOCKET PROTOCOL REFERENCE
================================================================================

Server -> Client message types:
- token: { type: "token", content: string }
- tool_start: { type: "tool_start", tool_name: string, message: string }
- tool_end: { type: "tool_end", tool_name: string }
- interrupt: { type: "interrupt", session_id, action, tool_name, preview, arguments }
- complete: { type: "complete", session_id: string }
- approval_ack: { type: "approval_ack", session_id, decision }
- error: { type: "error", message: string, pending_tool?: string }
- pong: { type: "pong" }
- complex_query_notice: { type: "complex_query_notice", message: string }  <- NEW
- large_data_notice: { type: "large_data_notice", message: string }        <- NEW
